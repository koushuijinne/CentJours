# ADR-008: 历史事件扩充至 100+ 条

## 状态: 已采纳

## 背景

`src/data/events/historical.json` 现有 33 条历史事件，覆盖百日王朝三条主线（北上、政治、战役）。数量不足以支撑 100 天游戏跨度，且所有事件权重相同。需扩充至 100+ 条并建立分级机制。

---

## 一、写什么 — 数据结构与内容分布

### 1.1 三级事件体系

| 级别 | `tier` 值 | 目标数量 | 叙事段数 | 数值影响 | 前端展示 |
|------|----------|---------|---------|---------|---------|
| 重大 | `major` | 15–20 | 3–5 段 | 大幅改变局势，可触发分支 | 全屏演出 |
| 普通 | `normal` | 40–50 | 2–3 段 | 中等影响，微调数值 | 侧边通知 |
| 微小 | `minor` | 30–40 | 1–2 段 | ±1~2 或无数值影响 | 日志滚动 |

JSON 示例：

```json
{
  "id": "ney_defection",
  "tier": "major",
  "label": "内伊倒戈",
  "day_range": [5, 7]
}
```

### 1.2 时间分布

事件密度反映百日的真实节奏，不均匀分布：

| 阶段 | 天数 | 密度 | 说明 |
|------|------|------|------|
| 北上进军 | Day 1–20 | 高 | 几乎每天有事，军心剧变 |
| 巴黎执政 | Day 20–50 | 中 | 政治博弈为主，节奏放缓 |
| 备战期 | Day 50–85 | 低 | 故意稀疏，逼迫玩家主动行动 |
| 滑铁卢战役 | Day 85–100 | 极高 | 3–4 天内决定一切 |

### 1.3 五种事件类型

| 类型 | 说明 | 典型 tier | 数量预算 |
|------|------|----------|---------|
| **真实历史** | 可查证的史实 | major / normal | 占总量 50–60% |
| **半虚构** | 家书、咖啡馆争论、新兵日记、市场物价 | minor 为主 | 占总量 30–35% |
| **联军视角** | "情报式"呈现，信息可能不完整甚至有误 | normal / minor | 10–15 条 |
| **玩家决策** | 重大事件附带选择分支（结构预留） | major | Rust 侧 `choices: Option<Vec<EventChoice>>`，JSON 暂不填充 |
| **司汤达日记** | NPC 叙事者的个人观察，连接 Rouge/Noir 调性 | minor | 穿插于各阶段 |

## 二、怎么写 — 逐条事件的写作 Checklist

> 每写一条事件，按以下清单逐项检查。不通过则不入库。

### 2.1 通用规则（所有事件必须遵守）

- [ ] **tier 与段数匹配**：major 3–5 段 / normal 2–3 段 / minor 1–2 段
- [ ] **day_range 落在正确阶段**：对照 §1.2 时间分布表
- [ ] **不编造可查证的历史事实**：人名、地名、日期、战果必须可考据；不确定则标注 `"fictional": true`

### 2.2 按类型的附加规则

#### 真实历史事件

- [ ] 客观史书笔法：有修辞但不散文化，一段话说清"发生了什么 + 为什么重要"
- [ ] major 事件须包含感官细节（声音、气味、天气、人群反应）
- [ ] normal 事件简洁有力，一句核心 + 一句影响

#### 半虚构事件

- [ ] 定位"无名者视角"：士兵、平民、家属，不用全知叙事
- [ ] 作为大事件的余波出现，不喧宾夺主
- [ ] 散文化写法允许，但须克制——《王氏之死》式一笔带过优于铺陈
- [ ] 不与同日的真实历史事件矛盾

#### 联军视角事件

- [ ] 以"截获情报""外交密函""前线侦察"等形式呈现
- [ ] 信息允许不完整甚至有误（战争迷雾）
- [ ] 集中在关键决策节点：维也纳外交、威灵顿/布吕歇尔部署、联军内部分歧

#### 司汤达日记

> TODO(history): 当前这一节的“司汤达在场观察者”设定需要改成 `Henri Gatien Bertrand` 宫廷总管日记。Stendhal 只保留为 Rouge/Noir 文学调性参考，不继续承担百日王朝在场叙事者身份。

- [ ] 第一人称，冷峻内省，不煽情
- [ ] 观察而非判断——记录所见，留白让玩家自己感受

#### 玩家直面文本的拿破仑第一人称

- [ ] 适用范围只限于玩家直面的主观文本：前 10 天教学、当日行动建议、三日节奏提示、阶段复盘、结局前短独白
- [ ] 语气要像“在下判断、在下回忆、在下命令”，不是现代口语，也不是煽情独白
- [ ] 句子长度偏短，重判断、节奏和代价，不堆抽象抒情
- [ ] 凡是涉及联军部署、议会动向、外交密电、外部观察者评论，默认不用拿破仑第一人称
- [ ] `historical_note` 默认保持第三人称或档案体，避免把可考据事实写成主观回忆
- [ ] 若一条文本需要同时承担“史实说明”和“玩家主观代入”，拆成两层：正文可主观，史注保持客观

### 2.3 文学质量底线

- [ ] 没有"据说""众所周知"等填充词
- [ ] 没有现代网络用语或穿越感表达
- [ ] 中文行文自然，不是英文直译腔
- [ ] major 事件读完有画面感；minor 事件读完有情绪残留

## 三、参考什么 — 按事件类型索引的书目

> 按"我在写哪种事件"查表，直接找到对应参考。

### 3.1 项目设计 DNA

> 这三本直接影响游戏核心机制，在 `docs/plans/product_plan.md` 中有明确关联。写任何事件前应理解它们的精神。

| 书名 | 作者 | 项目关联 |
|------|------|---------|
| 《红与黑》 *Le Rouge et le Noir* | 司汤达 | Rouge/Noir 双极系统命名源头；司汤达 NPC 日记的文学调性锚点 |
| 《战争与和平》 *War and Peace* | 托尔斯泰 | 命令偏差系统的思想来源；战场混沌 → major 战役事件中"信息不完整"的叙事策略 |
| 《悲惨世界》第二卷 *Les Misérables* | 雨果 | 全景→单兵的镜头切换手法；"命运转折点"的戏剧化处理 → 玩家决策分支叙事 |

### 3.2 写 major 真实历史事件时参考

> 关键词：多视角、感官重建、节奏控制

| 书名 | 作者 | 借鉴点 |
|------|------|--------|
| *Waterloo: Four Days, Three Armies, Three Battles* | Bernard Cornwell | 滑铁卢多视角战场写法，Day 85–100 直接参考 |
| *Les Cent-Jours* | Dominique de Villepin | 法语第一手政治叙事，提炼 major 政治事件措辞 |
| 《战斗的面目》 *The Face of Battle* | John Keegan | 从士兵身体感受（声音、气味、恐惧）写战斗 |
| 《最长的一天》 *The Longest Day* | Cornelius Ryan | 单日多视角切换：将军/士兵/平民/敌方 |
| 《八月炮火》 *The Guns of August* | Barbara Tuchman | 逐日推进的节奏控制——"高密度阶段每天有事" |

### 3.3 写 normal 政治/军事事件时参考

> 关键词：简洁有力、政治博弈、史书笔法

| 书名 | 作者 | 借鉴点 |

| *Napoleon: A Life* | Andrew Roberts | 全景传记，政治博弈细节丰富 |
| 《狼厅》 *Wolf Hall* | Hilary Mantel | "贴身第三人称"写权力博弈的日常质感，适合 Day 20–50 |

### 3.4 写 minor 半虚构事件时参考

| 《奶酪与蛆虫》 *The Cheese and the Worms* | Carlo Ginzburg | "从档案缝隙还原内心"——写士兵日记、新兵独白 |
| 《马丁·盖尔归来》 *The Return of Martin Guerre* | Natalie Zemon Davis | 单个小事件撬动大背景——写咖啡馆争论、家书 |
| 《王氏之死》 *The Death of Woman Wang* | Jonathan D. Spence | 极简笔触、克制情感——minor 级一句话事件的调性 |
| 《长安十二时辰》 | 马伯庸 | 紧迫时间 + 小人物推动大事件 |

### 3.5 写联军视角 / 情报式事件时参考

| 书名 | 作者 | 借鉴点 |

> 关键词：口述实录、去英雄化、家属视角

| 书名 | 作者 | 借鉴点 |

> 关键词：命令口吻、自我辩护、政治判断、情感克制

| 书名 | 借鉴点 |
|------|--------|
| 《拿破仑书信集》 | 句式短、判断快，适合写命令感和临场决断 |
| 《圣赫勒拿回忆录》 | 自我解释与历史回顾的语气，适合结局前独白与复盘 |
| 《拿破仑战报》 | 战况摘要、成果表述、行动优先级，适合行动建议与战役结算口吻 |

| 《拿破仑致约瑟芬情书》 | 只借极少量私人语感，用于罕见的疲惫、焦躁、眷恋时刻，不能泛化到所有文本 |

## 四、实施计划

1. JSON 结构新增 `tier` 字段，给现有 33 条事件分级
2. 分阶段扩充至 100+ 条，按 §1.2 时间阶段批量编写
3. 每条事件入库前过 §2 Checklist
4. Rust `HistoricalEvent` 结构体同步新增 `tier` 字段
5. 前端根据 `tier` 区分展示方式（后续迭代）
6. 玩家直面教学 / 建议 / 复盘文本可逐步引入拿破仑第一人称，但 `historical_note` 与外部事件保持客观层

## 后果

- 玩家体验从"线性叙事"转向"有节奏的策略决策"
- 维护成本增加：100+ 条事件的叙事质量需持续审查，Checklist 是质量底线
- 分级机制为前端差异化展示提供数据基础
- 书目按事件类型索引，写文案时直接查表，降低调性漂移风险
