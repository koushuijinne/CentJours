# ADR-008: 历史事件扩充至 100+ 条 — 分级体系与设计规范

## 状态: 已采纳

## 背景

当前 `src/data/events/historical.json` 包含 33 条历史事件，覆盖百日王朝三条主线（北上、政治、战役）。事件数量不足以支撑 100 天的游戏跨度，且所有事件权重相同，缺乏轻重之分。需要扩充至 100+ 条并建立分级机制。

## 决策

### 1. 三级事件体系

| 级别 | 英文标识 | 目标数量 | 叙事段数 | 数值影响 | 前端展示 |
|------|---------|---------|---------|---------|---------|
| **重大** | `major` | 15–20 | 3–5 段 | 大幅改变局势，可触发分支 | 全屏演出 |
| **普通** | `normal` | 40–50 | 2–3 段 | 中等影响，微调数值 | 侧边通知 |
| **微小** | `minor` | 30–40 | 1–2 段 | ±1~2 或无数值影响 | 日志滚动 |

JSON 新增 `tier` 字段：

```json
{
  "id": "ney_defection",
  "tier": "major",
  "label": "内伊倒戈",
  "day_range": [5, 7],
  ...
}
```

### 2. 叙事风格规范

- **真实历史事件**：客观史书笔法，参考《罗马帝国衰亡史》等通俗史书风格。略有修辞但不散文化，降低阅读门槛，让读者迅速抓到重点。
- **半虚构事件**：允许散文化写法，但须把控文学质量。定位为"无名者视角"——用士兵、平民、家属的视角补充大事件的余波，不编造历史事实。
- **分级对应**：重大事件必须精雕；普通事件简洁有力；微小事件一笔带过。

### 3. 时间分布：模拟历史节奏

事件密度不应均匀分布，应反映百日的真实节奏：

| 阶段 | 天数范围 | 密度 | 说明 |
|------|---------|------|------|
| 北上进军 | Day 1–20 | 高 | 几乎每天有事，军心剧变 |
| 巴黎执政 | Day 20–50 | 中 | 政治博弈为主，节奏放缓 |
| 备战期 | Day 50–85 | 低 | 故意稀疏，逼迫玩家主动行动 |
| 滑铁卢战役 | Day 85–100 | 极高 | 3–4 天内决定一切 |

### 4. 半虚构事件规范

- 允许"半虚构"事件填充日常氛围
- 不得编造可被查证的历史事实
- 典型形式：家书、咖啡馆争论、新兵日记、市场物价波动
- 半虚构事件主要集中在 `minor` 级别
- 作为大事件的余波出现，不喧宾夺主

### 5. 联军视角事件

- 数量有限（10–15 条），集中在关键决策节点
- 以"情报式"呈现——信息可能不完整甚至有误，符合战争迷雾设计
- 覆盖范围：维也纳外交决策、威灵顿/布吕歇尔军事部署、联军内部分歧

### 6. 重大事件预留玩家选择（结构预留）

重大事件未来应支持玩家决策分支（如内伊倒戈后的处置、附加法令的激进/保守版本、格鲁希分兵方向）。当前在 Rust 侧预留 `choices: Option<Vec<EventChoice>>` 字段，JSON 暂不填充，后续迭代实现。

## 实施计划

1. JSON 结构新增 `tier` 字段，给现有 33 条事件分级
2. 分阶段扩充至 100+ 条，按时间阶段批量编写
3. Rust `HistoricalEvent` 结构体同步新增 `tier` 字段
4. 前端根据 `tier` 区分展示方式（后续迭代）

## 后果

- 玩家体验从"线性叙事"转向"有节奏的策略决策"
- 维护成本增加：100+ 条事件的叙事质量需持续审查
- 分级机制为前端差异化展示提供数据基础
