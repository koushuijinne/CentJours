## 目标

把真人试玩暴露出的核心体验问题转成可执行、可验证、可排期的修复任务，不直接改写原始反馈文档。

## 核心问题

1. 每天只能执行一个政策，行动经济过窄，回合体验单薄。
2. 教程和事件停留在侧栏文本，缺少弹窗打断感和可回看入口。
3. 玩家看不到结局目标和达成条件。
4. 当前 UI 以大面板为主，地图被压缩，缺少 P 社式的地图优先结构。
5. 当前中文试玩阶段仍残留英文 UI，语言策略不统一。

## 修复计划

### P0-1 行动经济重构

- 目标：把当前“一天一张牌”改成“至少一次行军 + 2 到 3 次决策”的日内行动节奏。
- 最小落地：
  - 引入每日行动点或分栏行动槽。
  - 把行军从普通政策中解耦，单独成为当天基础动作。
  - 政策牌改为消耗独立决策点，允许同日执行多次。
- 验证：
  - Rust 集成测试覆盖每日行动数、行军与决策的结算边界。
  - `GdUnit4` 覆盖日内多动作提交、次日推进、UI 剩余行动数同步。

### P0-2 教程 / 事件弹窗化 + 日志回看

- 目标：把新手教学、历史事件、关键提示改成弹窗呈现，关闭后进入日志，允许重复打开。
- 最小落地：
  - 新增统一弹窗模型：标题、正文、按钮、来源标签。
  - `History & Narrative` 面板改成可回看入口，而不只是静态文本容器。
  - 前 10 天教程从侧栏提示改成阶段性弹窗。
- 验证：
  - `GdUnit4` 覆盖弹窗打开、关闭、重复打开、日志回看。
  - Windows 真机验收弹窗尺寸、滚动、中文换行。

### P0-3 结局目标入口

- 目标：在主界面提供一个可点击入口，明确玩家可达成的结局类型和条件方向。
- 最小落地：
  - 顶栏或侧栏新增“结局 / 战略目标”按钮。
  - 展示已实现结局路径、触发倾向和当前局势关联。
  - 区分“已知条件”和“隐藏条件”，避免剧透过度。
- 验证：
  - `GdUnit4` 覆盖按钮入口、弹窗内容、关闭恢复。
  - Rust / GDScript 契约校验结局枚举与 UI 文本一致。

### P0-4 中文优先收口

- 目标：中文试玩阶段界面和关键文本全中文，英文只保留技术标识。
- 最小落地：
  - 清理主菜单、弹窗、教程、结局、地图详情中的残留英文。
  - 保持代码标识英文，玩家可见文本中文。
- 验证：
  - 手工清单核对主要 UI 区域无残留英文。
  - 后续再做 i18n，不在本轮引入双语混排。

### P1-1 地图优先布局重构

- 目标：对齐参考图，把地图作为主视觉，详情面板改成可展开、可关闭、可收纳。
- 最小落地：
  - 压缩固定侧栏面积，提高地图可见区域。
  - 详情入口改为图标按钮或二级面板，默认收起。
  - 保留缩放与 hover / click 详情，但避免长期遮挡地图。
- 验证：
  - Windows 真机验收地图占比、节点可读性、面板遮挡。
  - `GdUnit4` 覆盖面板展开 / 收起和状态恢复。

## 任务拆分建议

1. 先做 `P0-1`，因为它直接决定一局是否“有事可做”。
2. 再做 `P0-2` 和 `P0-3`，补上引导与目标感。
3. 然后做 `P0-4`，统一中文试玩体验。
4. 最后做 `P1-1`，把界面从开发态收口到产品态。
