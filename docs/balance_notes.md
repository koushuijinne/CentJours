## M0 阶段（2026-03-17）— 初版问题记录

**测试工具**: `tests/monte_carlo_balance.py` (1000局/策略)

### 问题1：纯军事策略导致政治崩溃（92.5%）

**原因**: 军事政策（征兵令/增加军费）累积惩罚自由派和民众，而自然恢复速度不足以抵消。

**方向**: 军事胜利应该为所有派系带来更大的支持度提升（历史上"帝国万岁"效应）；自由派惩罚应设上限。

### 问题2：政治/均衡策略胜率100%

**原因**: Day 80+的反法同盟兵力不够强大（`50000 + coalition_strength * 100`），而Napoleon有12万部队时战斗模型必胜。

**方向**: 滑铁卢特殊处理——即使Napoleon赢得大多数战斗，Day 100也应该有威灵顿+布吕歇尔联合的不确定性；需要引入"单场决定命运"机制。

### 问题3：策略差异过于极端

**方向**: 目标是各策略的胜率均在15%-35%之间，政治崩溃率在10%-25%之间。玩家需要强迫在军政之间做出tradeoff。

## 核心公式（M6 调参直接查阅）

### 决战联军兵力（Day 80+）
```
enemy_troops = 70_000 + coalition_strength × 1_400
```
叠加 ±25% Waterloo Chaos 随机因子（`rng.gen_range(-0.25..=0.25)`）

### 早期战斗联军兵力
```
enemy_troops = 8_000 + coalition_strength × 20
```

### coalition_strength 累积速率
| 日期段 | 每日递增 |
|--------|---------|
| Day 1–19 | +0.0 |
| Day 20–29 | +0.3 |
| Day 30–59 | +1.0 |
| Day 60–100 | +2.0 |

## 胜利条件

| 条件 | 描述 |
|------|------|
| 标准胜利 | Day 100 结束：`victories >= 5 AND legitimacy >= 45.0` |
| 即时胜利 | 决战胜利（Day 80+）且上述条件同时满足 → 立即 `NapoleonVictory` |
| 历史结局 | `victories >= 3 AND legitimacy >= 35.0` → `WaterlooHistorical` |
| 流放结局 | 其余情况 → `WaterlooDefeat` |
| 政治崩溃 | 任意两派系 < 10 → `PoliticalCollapse`（随时触发） |
| 军事崩溃 | `total_troops < 5_000` → `MilitaryAnnihilation` |

## 调参参考

| 问题 | 调整方向 |
|------|---------|
| 胜率 < 15% | 降低 `coalition_strength` 累积速率，或提高战胜合法性奖励 |
| 胜率 > 35% | 扩大 waterloo_chaos 范围，或提高决战基础兵力 |
| 政治崩溃率 > 30% | 降低政策惩罚强度，或提高 `daily_tick` 恢复速率 |
| 军事崩溃率过高 | 检查 `casualty_rates()` 中的损耗系数 |
