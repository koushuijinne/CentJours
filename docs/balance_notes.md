# 平衡性测试记录 — M0阶段

**测试日期**: 2026-03-17
**测试工具**: `tests/monte_carlo_balance.py` (1000局/策略)

## 发现的平衡问题

### 问题1：纯军事策略导致政治崩溃（92.5%）

**原因**: 军事政策（征兵令/增加军费）累积惩罚自由派和民众，而自然恢复速度不足以抵消。

**方向**: 军事胜利应该为所有派系带来更大的支持度提升（历史上"帝国万岁"效应）；自由派惩罚应设上限。

### 问题2：政治/均衡策略胜率100%

**原因**: Day 80+的反法同盟兵力不够强大（`50000 + coalition_strength * 100`），而Napoleon有12万部队时战斗模型必胜。

**方向**: 滑铁卢特殊处理——即使Napoleon赢得大多数战斗，Day 100也应该有威灵顿+布吕歇尔联合的不确定性；需要引入"单场决定命运"机制。

### 问题3：策略差异过于极端

**方向**: 目标是各策略的胜率均在15%-35%之间，政治崩溃率在10%-25%之间。玩家需要强迫在军政之间做出tradeoff。

## 下一步调整（M1阶段）

- [ ] 军事胜利对民众/自由派的波动效果调整：+8（现在是+3）
- [ ] 反法同盟Day 80后集结速度加倍
- [ ] 增加"滑铁卢特殊规则"：无论兵力，该战役额外随机因子±25%（代表战场迷雾）
- [ ] 将自由派/民众惩罚类政策的上限设为20%（而不是无限叠加）
- [ ] 添加"命令偏差"对战斗结果的影响（目前战斗模型未引入将领偏差）
